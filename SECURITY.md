# Security Policy

## Supported Versions

The project is preparing its first public beta. Security fixes target the latest released version once releases begin.

## Reporting a Vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Contact placeholder: `SECURITY_CONTACT_TBD`

Until a real contact is configured, report privately to the repository owner through GitHub.

## Sensitive Data

NB Capture can record the screen, microphone, and webcam. Treat generated captures as potentially sensitive.

Do not attach private screenshots, videos, logs, `.env` files, credentials, certificates, or personal data to public issues.

## Maintainer Checklist

- Enable GitHub secret scanning.
- Enable push protection.
- Enable Dependabot alerts and security updates.
- Enable CodeQL/code scanning.
- Require CI before merging protected branches.
